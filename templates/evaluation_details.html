The evaluator uses the following fixed setup:

  • Environment: MiniGrid-Empty-5x5-v0
  • Observation fields: image, direction, mission
  • Action space: discrete with n=7
  • Per-episode step budget: 100

Each evaluation step sends the current observation to:

POST {agent_url}/act

Request body:

{
  "phase_codename": "dev|test",
  "env_id": "MiniGrid-Empty-5x5-v0",
  "episode_index": 0,
  "episode_seed": 0,
  "step_index": 0,
  "max_steps": 100,
  "action_space": {"type": "discrete", "n": 7},
  "observation": {
    "image": [[[...]]],
    "direction": 0,
    "mission": "get to the green goal square"
  }
}

Expected response:

{"action": 0}

Any timeout, non-2xx response, invalid JSON, missing action, or out-of-range action causes the submission to fail.

EvalAI leaderboard sync currently uses this placeholder metric contract: BinFill, PickXtimes, SwingXtimes, StopCube, VideoUnmask, VideoUnmaskSwap, ButtonUnmask, ButtonUnmaskSwap, PickHighlight, VideoRepick, VideoPlaceButton, VideoPlaceOrder, MoveCube, InsertPeg, PatternLock, RouteStick, SuccessRate, and OverallSuccessRate.

Real per-task scoring is not wired in this revision. Every leaderboard column is currently returned as a placeholder 0.0 value so the EvalAI configuration and remote evaluation payloads remain consistent.
